    Create a reusable confirmation dialog component to replace `window.confirm()` calls. This is a simple modal overlay pattern.

    **ConfirmDialog.tsx:**

    ```typescript
    import './ConfirmDialog.css';

    interface ConfirmDialogProps {
      open: boolean;
      title: string;
      message: string;
      confirmLabel?: string;
      cancelLabel?: string;
      onConfirm: () => void;
      onCancel: () => void;
    }

    export function ConfirmDialog({
      open,
      title,
      message,
      confirmLabel = 'Confirm',
      cancelLabel = 'Cancel',
      onConfirm,
      onCancel,
    }: ConfirmDialogProps) {
      if (!open) return null;

      return (
        <div className="confirm-dialog-overlay" onClick={onCancel} role="presentation">
          <div
            className="confirm-dialog"
            role="alertdialog"
            aria-modal="true"
            aria-labelledby="confirm-dialog-title"
            aria-describedby="confirm-dialog-message"
            onClick={(e) => e.stopPropagation()}
          >
            <h3 id="confirm-dialog-title" className="confirm-dialog-title">{title}</h3>
            <p id="confirm-dialog-message" className="confirm-dialog-message">{message}</p>
            <div className="confirm-dialog-actions">
              <button
                className="confirm-dialog-btn confirm-dialog-btn--cancel"
                onClick={onCancel}
              >
                {cancelLabel}
              </button>
              <button
                className="confirm-dialog-btn confirm-dialog-btn--confirm"
                onClick={onConfirm}
                autoFocus
              >
                {confirmLabel}
              </button>
            </div>
          </div>
        </div>
      );
    }
    ```

    Add a ref to track the previous message count:

    Add a `useEffect` after the existing scroll effects:

    ```typescript
    // Track new messages arriving while scrolled up
    useEffect(() => {
      const currentCount = messages.length;
      const prevCount = prevMessageCount.current;
      if (currentCount > prevCount && !isAtBottom) {
        setNewCount((n) => n + (currentCount - prevCount));
      }
      prevMessageCount.current = currentCount;
    }, [messages.length, isAtBottom]);
    ```

    Also reset `prevMessageCount` when the channel changes:

    In the existing `useEffect` for channel change (the one that scrolls to bottom on initial load), add:
    ```typescript
    prevMessageCount.current = messages.length;
    ```

    Wait — the existing effect uses `loading` and `channelId` as deps, which fires before messages are loaded. Better to add a separate effect for channel change reset:

    ```typescript
    // Reset new message count on channel change
    useEffect(() => {
      setNewCount(0);
      prevMessageCount.current = 0;
    }, [channelId]);
    ```

    And update the existing initial load effect to also reset prevMessageCount:
    ```typescript
    useEffect(() => {
      if (!loading && listRef.current) {
        listRef.current.scrollTop = listRef.current.scrollHeight;

        prevMessageCount.current = messages.length;

    }, [loading, channelId, messages.length]);
    ```

    Note: Added `messages.length` to the deps so prevMessageCount is accurate after initial load.
