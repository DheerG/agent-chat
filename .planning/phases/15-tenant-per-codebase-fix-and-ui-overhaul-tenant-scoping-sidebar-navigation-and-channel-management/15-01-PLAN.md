Output: Modified TeamInboxWatcher.processTeam() with cwd extraction, plus tests for the fix.

Changes needed:

1. After reading the team config, extract the codebase path from the first member's `cwd` field:
```typescript
// Extract actual codebase path from team members
const codebasePath = config.members?.find(m => m.cwd)?.cwd ?? teamPath;
```

2. Use `basename(codebasePath)` as the tenant name instead of teamName for human readability:
```typescript
const { basename } = require('path'); // already imported at top
const tenantName = basename(codebasePath);
```

Note: `basename` is already imported from 'path' at line 2.

3. Change the upsertByCodebasePath call:
```typescript
const tenant = await this.services.tenants.upsertByCodebasePath(
  tenantName,
  codebasePath,
);
```

4. Keep the channel name as `teamName` (each team gets its own channel within the shared tenant):
```typescript
let channel = existingChannels.find(c => c.name === teamName);
if (!channel) {
  channel = await this.services.channels.create(tenant.id, {
    name: teamName,
    type: 'manual',
  });
}
```

This ensures:
- Multiple teams on the same codebase share one tenant
- Each team has its own channel within that tenant
- The tenant name is human-readable (e.g., "agent-chat" not a UUID or full path)
