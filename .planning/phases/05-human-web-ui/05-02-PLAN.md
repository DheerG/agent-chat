**packages/client/src/hooks/useWebSocket.ts:**

Create a hook that manages a single WebSocket connection per tenant.

```typescript
import { useState, useEffect, useRef, useCallback } from 'react';
import type { Message } from '@agent-chat/shared';

type MessageHandler = (message: Message) => void;

interface UseWebSocketOptions {
  tenantId: string | null;
  onMessage?: MessageHandler;
}

interface UseWebSocketReturn {
  connected: boolean;
  subscribe: (channelId: string, lastSeenId?: string) => void;
  unsubscribe: (channelId: string) => void;
}
```

Implementation details:
- Opens WebSocket to `ws://${window.location.host}/ws?tenantId=${tenantId}` when tenantId is set
- Tracks connected state (boolean)
- On `message` event from WS, parse JSON. If type === 'message', call `onMessage` callback with the message
- If type === 'catchup', call `onMessage` for each message in the batch
- subscribe() sends `{ type: 'subscribe', channelId, lastSeenId }` to server
- unsubscribe() sends `{ type: 'unsubscribe', channelId }` to server
- On close/error, attempt reconnect with exponential backoff (1s, 2s, 4s, max 30s)
- On reconnect, re-subscribe to all active channel subscriptions with lastSeenId
- Cleanup: close WebSocket on tenantId change or unmount
- Use useRef to track subscriptions and reconnect timer
- Use useCallback for stable subscribe/unsubscribe functions

**packages/client/src/hooks/useMessages.ts:**

Create a hook that fetches initial messages via REST and receives live messages via WebSocket.

```typescript
import { useState, useEffect, useCallback, useRef } from 'react';
import type { Message } from '@agent-chat/shared';
import { fetchMessages, sendMessage as sendMessageApi } from '../lib/api';

interface UseMessagesReturn {
  messages: Message[];
  loading: boolean;
  error: string | null;
  sendMessage: (content: string, parentMessageId?: string) => Promise<void>;
  addMessage: (message: Message) => void;
}
```

Implementation details:
- When tenantId/channelId change, fetch initial messages via REST (last 50)
- `addMessage(msg)` appends a new message to the list (called by WebSocket handler)
- Deduplication: check if message.id already exists before adding (WS might deliver a message we sent via REST)
- `sendMessage(content, parentMessageId?)` calls the REST API and does NOT add to list (the WS broadcast will deliver it back)
- Track loading and error states
- Return lastSeenId (the id of the most recent message) for WebSocket reconnect catch-up

**MessageFeed component:**

- Uses useMessages to fetch/manage messages
- Uses useWebSocket to connect and subscribe

- Empty state: "No messages yet" centered text

**MessageItem component:**
- Receives a Message object as prop
- Renders differently based on senderType and messageType:
  - senderType 'agent': left-aligned with colored initials avatar, agent-style background
  - senderType 'human': left-aligned with different colored avatar, human-style background
  - senderType 'system': centered, muted, no avatar
  - messageType 'event': delegates to EventCard component
- Shows sender name, relative timestamp ("2m ago", "1h ago")
- Thread indicator: if message has replies (need a replyCount or check), show "N replies" link
- Use a utility function for relative time formatting

**ComposeInput component:**
- Text input (textarea, 1 row, auto-grows)
- Send on Enter (without shift), Shift+Enter for newline
- Send button on the right
- Calls onSend(content) prop when submitted
- Clears input after send
- Disabled when no channelId selected
- Placeholder: "Type a message..."

**EventCard component:**
- Receives a Message with messageType 'event' as prop
- Collapsed by default: shows tool icon + tool name from metadata.toolName
- Click to expand: shows tool arguments (metadata.arguments) as formatted JSON
- If metadata.result exists, show result summary below arguments
- Subtle blue-gray background, rounded corners, smaller font than chat messages
- Toggle button (chevron or +/- indicator)

**CSS for all components:**
- MessageFeed: flex column, scrollable message area, compose pinned at bottom
- MessageItem: flex row with avatar + content, different backgrounds for agent/human
- ComposeInput: flex row with textarea + send button, border-top separator
- EventCard: collapsible card with transition, distinct background

All CSS uses plain CSS files (not modules) imported directly — keep it simple for a dev tool.
