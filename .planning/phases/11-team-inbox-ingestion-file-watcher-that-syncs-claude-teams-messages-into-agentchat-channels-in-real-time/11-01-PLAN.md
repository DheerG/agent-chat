  <name>Task 1: Create TeamInboxWatcher service</name>
  <files>packages/server/src/watcher/TeamInboxWatcher.ts, packages/server/src/watcher/index.ts</files>
  <action>
Create the TeamInboxWatcher class that:

1. **Constructor** takes `Services` and an optional `teamsDir` (default: `~/.claude/teams/`).

2. **start()** method:
   - Scans `teamsDir` for existing team directories
   - For each team directory: reads `config.json`, creates tenant + channel, processes existing inbox files
   - Sets up `fs.watch` on `teamsDir` (recursive on macOS) to detect:
     - New team directories appearing
     - Inbox files being created/modified
   - Returns void (async)

3. **stop()** method:
   - Closes all fs.watch watchers
   - Clears internal state

4. **Team discovery and setup:**
   - When a team directory is found, read `config.json` to get team name and description
   - Call `services.tenants.upsertByCodebasePath(teamName, teamsDir + '/' + teamName)` to get/create tenant
   - Call `services.channels.create(tenantId, { name: teamName, type: 'manual' })` to get/create channel
   - Cache the `tenantId` and `channelId` per team name to avoid repeated lookups
   - Handle case where channel already exists (check `listByTenant` first)

5. **Inbox processing:**
   - Read each inbox JSON file (`inboxes/{agent-name}.json`)
   - Parse as JSON array of inbox messages
   - Track last-processed index per inbox file path (Map<string, number>) to only process new messages
   - For EACH message in the inbox, construct a dedup key: `${msg.from}|${msg.timestamp}|${hashText(msg.text)}`
   - Use a `Set<string>` of dedup keys to skip already-ingested messages (handles broadcasts)
   - For new messages, call `services.messages.send(tenantId, { ... })` with:
     - channelId: the team channel
     - senderId: `${msg.from}@${teamName}` (unique per team)
     - senderName: msg.from (e.g., "principal-engineer")
     - senderType: 'agent'
     - content: msg.text
     - messageType: 'text' (for regular messages) or 'event' (for structured JSON like idle_notification)
     - metadata: { recipient: inboxOwnerName, color: msg.color, summary: msg.summary, read: msg.read, is_broadcast: (seen in multiple inboxes), original_type: (parsed from JSON text if applicable) }

6. **Structured message detection:**
   - Try to parse `msg.text` as JSON
   - If it has a `type` field (e.g., "idle_notification", "shutdown_request"), set messageType to 'event' and store `original_type` in metadata
   - If not parseable as JSON, it's a regular text message

7. **File change handling:**
   - On fs.watch event for an inbox file: debounce by 100ms, then re-read the full file, diff against last-processed index, ingest new messages
   - On fs.watch event for a new directory in teamsDir: check if it's a team (has config.json), set up watching
   - Use `createHash('sha256').update(text).digest('hex').slice(0, 16)` for text hashing (16 chars is sufficient)

8. **Error handling:**
   - Wrap all JSON.parse calls in try/catch — log and skip on invalid JSON (file may be mid-write)
   - Wrap fs.watch setup in try/catch — log if directory doesn't exist
   - Handle ENOENT (file deleted) gracefully
   - Never throw from event handlers — log errors and continue

Create `packages/server/src/watcher/index.ts` that exports `TeamInboxWatcher`.
