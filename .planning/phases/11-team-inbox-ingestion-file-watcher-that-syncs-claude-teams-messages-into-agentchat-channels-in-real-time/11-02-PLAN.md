1. Import `TeamInboxWatcher` from `./watcher/index.js`

2. After creating services and before starting the HTTP server, create and start the watcher:
   ```
   const teamsDir = process.env['TEAMS_DIR'] ?? path.join(os.homedir(), '.claude', 'teams');
   const watcher = new TeamInboxWatcher(services, teamsDir);
   watcher.start().then(() => {
     console.log(JSON.stringify({ event: 'team_inbox_watcher_started', teamsDir }));
   }).catch((err) => {
     console.error(JSON.stringify({ event: 'team_inbox_watcher_error', error: String(err) }));
   });
   ```

3. Add `import * as os from 'os'` and `import * as path from 'path'` at the top (or use the existing Node.js style imports).

4. In the SIGTERM handler, BEFORE closing WebSocket connections, stop the watcher:
   ```
   watcher.stop();
   ```

   This should happen first in the shutdown sequence since the watcher generates writes — stop it before draining the write queue.

5. The updated shutdown order should be:
   1. Stop team inbox watcher (stop generating new writes)
   2. Close WebSocket connections
   3. Close HTTP server
   4. Drain write queue
   5. Close database

Maintain the existing code style: structured JSON logging, same import patterns.
