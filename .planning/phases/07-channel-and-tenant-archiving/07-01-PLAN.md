This is a nullable column -- NULL means active, ISO 8601 string means archived. Update the Drizzle-inferred types at the bottom (TenantRow, ChannelRow will automatically include archivedAt).

    2. **Types (packages/shared/src/types.ts):** Add `archivedAt: string | null;` field to both `Tenant` and `Channel` interfaces. Add it after `createdAt` for Tenant, and after `updatedAt` for Channel.

    3. **Migration (packages/server/src/db/index.ts):** After the `rawDb.exec(CREATE_TABLES_SQL)` call, add two try-catch blocks to idempotently add the archived_at columns:
    ```typescript
    try { rawDb.exec('ALTER TABLE tenants ADD COLUMN archived_at TEXT'); } catch { /* column exists */ }
    try { rawDb.exec('ALTER TABLE channels ADD COLUMN archived_at TEXT'); } catch { /* column exists */ }
    ```
    SQLite does NOT support `IF NOT EXISTS` for ALTER TABLE ADD COLUMN, so try-catch is required. This pattern runs on every startup and silently succeeds if columns already exist.

    4. **Tenant queries (packages/server/src/db/queries/tenants.ts):**
    - Import `isNull`, `isNotNull` from `drizzle-orm` (add to existing `eq` import)
    - Update `rowToTenant()` to include `archivedAt: row.archivedAt ?? null`
    - Update `listAll()` to add `.where(isNull(tenants.archivedAt))` filter -- this is CRITICAL to prevent archived tenants from appearing in the active sidebar
    - Add `listArchived()`: same as listAll but with `.where(isNotNull(tenants.archivedAt))` filter
    - Add `async archiveTenant(id: string): Promise<boolean>`: uses `queue.enqueue()` to update tenant archivedAt to `new Date().toISOString()`, returns whether changes > 0
    - Add `async restoreTenant(id: string): Promise<boolean>`: uses `queue.enqueue()` to set tenant archivedAt to null, returns whether changes > 0
    - Note: Tenant archive/restore does NOT cascade to channels here -- that's handled at the service layer which calls both tenant and channel queries in sequence

    5. **Channel queries (packages/server/src/db/queries/channels.ts):**
    - Import `isNull`, `isNotNull` from `drizzle-orm`
    - Update `rowToChannel()` to include `archivedAt: row.archivedAt ?? null`
    - Update `getChannelsByTenant()` to add `isNull(channels.archivedAt)` to the where clause (use `and()` with existing `eq(channels.tenantId, tenantId)`)
    - Add `getArchivedChannelsByTenant(tenantId: string): Channel[]`: same as getChannelsByTenant but with `isNotNull(channels.archivedAt)`
    - Add `async archiveChannel(tenantId: string, channelId: string): Promise<boolean>`: set archivedAt to now, where id matches AND tenantId matches AND archivedAt IS NULL (prevents double-archive). Return changes > 0.
    - Add `async restoreChannel(tenantId: string, channelId: string): Promise<boolean>`: set archivedAt to null, where id matches AND tenantId matches AND archivedAt IS NOT NULL. Return changes > 0.
    - Add `async archiveChannelsByTenant(tenantId: string): Promise<void>`: set archivedAt to now on ALL channels for the tenant (used by tenant archive cascade)
    - Add `async restoreChannelsByTenant(tenantId: string): Promise<void>`: clear archivedAt on ALL channels for the tenant (used by tenant restore cascade)

    IMPORTANT: The `insertTenant()` return and `insertChannel()` return objects must also include `archivedAt: null` since the types now require it.
