With a type-aware CASE expression:

```sql
END)
```

```typescript
getActiveChannelsByTenant(tenantId: string): Channel[] {
  const rows = rawDb.prepare(
    `SELECT c.id, c.tenant_id, c.name, c.session_id, c.type, c.created_at, c.updated_at, c.archived_at, c.user_archived
     FROM channels c
     LEFT JOIN (
       SELECT channel_id, MAX(created_at) as last_activity
       FROM messages
       GROUP BY channel_id
     ) m ON c.id = m.channel_id
     WHERE c.tenant_id = ? AND c.archived_at IS NULL
       AND (m.last_activity IS NOT NULL AND m.last_activity >= CASE c.type
         WHEN 'session' THEN datetime('now', '-8 hours')
         ELSE datetime('now', '-48 hours')
       END)
     ORDER BY c.name`
  ).all(tenantId) as ChannelRawRow[];
  return rows.map(rawRowToChannel);
},
```

  <name>Task 2: Update getChannelsByTenantWithStale with type-aware threshold</name>
  <files>packages/server/src/db/queries/channels.ts</files>
  <action>
Modify the `getChannelsByTenantWithStale` method to use a type-aware stale threshold in the CASE expression.

Replace the is_stale CASE:
```sql
CASE
  WHEN m.last_activity IS NULL THEN 1
  WHEN m.last_activity < datetime('now', '-48 hours') THEN 1
  ELSE 0
END as is_stale
```

With a type-aware version:
```sql
CASE
  WHEN m.last_activity IS NULL THEN 1
  WHEN c.type = 'session' AND m.last_activity < datetime('now', '-8 hours') THEN 1
  WHEN c.type != 'session' AND m.last_activity < datetime('now', '-48 hours') THEN 1
  ELSE 0
END as is_stale
```

The full query should read:
```typescript
getChannelsByTenantWithStale(tenantId: string): Array<Channel & { stale: boolean }> {
  interface ChannelWithStaleRow extends ChannelRawRow {
    is_stale: number;
  }
  const rows = rawDb.prepare(
    `SELECT c.id, c.tenant_id, c.name, c.session_id, c.type, c.created_at, c.updated_at, c.archived_at, c.user_archived,
       CASE
         WHEN m.last_activity IS NULL THEN 1
         WHEN c.type = 'session' AND m.last_activity < datetime('now', '-8 hours') THEN 1
         WHEN c.type != 'session' AND m.last_activity < datetime('now', '-48 hours') THEN 1
         ELSE 0
       END as is_stale
     FROM channels c
     LEFT JOIN (
       SELECT channel_id, MAX(created_at) as last_activity
       FROM messages
       GROUP BY channel_id
     ) m ON c.id = m.channel_id
     WHERE c.tenant_id = ? AND c.archived_at IS NULL
     ORDER BY c.name`
  ).all(tenantId) as ChannelWithStaleRow[];
  return rows.map(row => ({
    ...rawRowToChannel(row),
    stale: row.is_stale === 1,
  }));
},
```

To test the different thresholds, we need to insert messages with controlled timestamps directly into the database. The tests need to simulate:
1. A session channel with a message 4 hours old (within 8h = NOT stale)
2. A session channel with a message 10 hours old (beyond 8h = stale)
3. A manual channel with a message 10 hours old (within 48h = NOT stale)
4. A manual channel with a message 50 hours old (beyond 48h = stale)

Since we can't easily control message timestamps through the API (messages always get `now()` timestamps), we'll insert messages directly using raw SQL in the test. Access the database via the services/test setup.

```typescript
describe('Differentiated stale thresholds by channel type', () => {
  test('session channels go stale after 8 hours', async () => {
    const tenant = await createTenant('/stale-session-8h');

    const createRes = await app.request(`/api/tenants/${tenant.id}/channels`, {

      method: 'POST',

    const defaultRes = await app.request(`/api/tenants/${tenant.id}/channels`);
    const defaultBody = await defaultRes.json() as { channels: Channel[] };
    expect(defaultBody.channels.length).toBe(2);
  });
});
```
