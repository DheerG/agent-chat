5. Comprehensive tests for all scenarios

Add a new query method `getChannelsByNamePrefix` to the channel queries object in `packages/server/src/db/queries/channels.ts`.

This query finds all channels (active and archived) whose name exactly matches `baseName` or matches the pattern `baseName-N` (where N is a number). This is needed for channel name disambiguation when a different team session uses the same team name.

Add this method inside the return object of `createChannelQueries`, after the `getChannelByName` method (around line 103):

```typescript
    /** Find all channels with a given base name or disambiguated variants (name, name-2, name-3, etc.) */
    getChannelsByNamePrefix(tenantId: string, baseName: string): Channel[] {
      const rows = rawDb.prepare(
        `SELECT id, tenant_id, name, session_id, type, created_at, updated_at, archived_at, user_archived
         FROM channels
         WHERE tenant_id = ? AND (name = ? OR name GLOB ? || '-[0-9]*')
         ORDER BY name`
      ).all(tenantId, baseName, baseName) as ChannelRawRow[];
      return rows.map(rawRowToChannel);
    },
```

Note: Using SQLite GLOB pattern `name GLOB baseName || '-[0-9]*'` matches names like "eval-1663-2", "eval-1663-3" but NOT "eval-1663-alpha" or "eval-1663-2-extra". The `[0-9]*` means one or more digits.

IMPORTANT: The GLOB pattern needs to handle names that themselves contain dashes. For example, "eval-1663" should match "eval-1663" (exact) and "eval-1663-2" but not "eval-1663-extra". The query uses the exact baseName for exact match and GLOB for the suffix pattern. This works because GLOB is case-sensitive and the pattern is anchored by the parameter binding.
