<!-- Key integration points the executor needs to understand -->

Claude Code .claude/settings.json hook format:
```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "curl -s -X POST http://localhost:5555/api/hooks/SessionStart -H 'Content-Type: application/json' -d \"$(cat)\" 2>/dev/null || true"
          }
        ]
      }
    ]
  }
}
```

Claude Code .claude/settings.json MCP format:
```json
{
  "mcpServers": {
    "agent-chat": {
      "command": "node",
      "args": ["/absolute/path/to/packages/mcp/dist/index.js"],
      "env": {
        "AGENT_CHAT_DB_PATH": "/Users/username/.agent-chat/data.db",
        "AGENT_CHAT_TENANT_ID": "auto",
        "AGENT_CHAT_AGENT_NAME": "claude-agent",
        "AGENT_CHAT_CWD": "/absolute/path/to/target-project"
      }
    }
  }
}
```

AgentChat server health endpoint:
- GET http://localhost:5555/health → { "status": "ok", "timestamp": "..." }

AgentChat hook endpoint:
- POST http://localhost:5555/api/hooks/:eventType
- Body: JSON with session_id, cwd, and event-specific fields
- Events: SessionStart, SessionEnd, PreToolUse, PostToolUse

MCP server binary:
- packages/mcp/dist/index.js (must be built via pnpm build)
- Environment variables: AGENT_CHAT_DB_PATH, AGENT_CHAT_TENANT_ID, AGENT_CHAT_AGENT_ID, AGENT_CHAT_AGENT_NAME, AGENT_CHAT_CWD

Default DB path: ~/.agent-chat/data.db (from packages/server/src/db/config.ts)
Default server port: 5555 (from packages/server/src/index.ts)

<task type="auto">
  <name>Task 1: Create the JSON merge helper (scripts/lib/merge-settings.cjs)</name>
  <files>
    scripts/lib/merge-settings.cjs
  </files>
  <behavior>
    - When run with --mode=setup, merges AgentChat hooks and mcpServers into a target settings.json
    - When run with --mode=teardown, removes AgentChat entries from a target settings.json
    - When target file doesn't exist (setup mode), creates it with just AgentChat config
    - When target file has existing hooks, appends AgentChat hooks without duplicating
    - When target file has existing mcpServers, adds agent-chat key alongside others
    - When run twice (idempotent), doesn't create duplicate entries
    - When run with --mode=teardown, only removes AgentChat entries, preserves everything else
    - When run with --test, runs built-in self-tests and exits with code 0 on pass
    - Accepts --target, --agent-chat-dir, --project-dir arguments
  </behavior>
  <action>
    1. Create `scripts/lib/` directory.

    2. Create `scripts/lib/merge-settings.cjs` as a CommonJS Node.js script (no build step needed).

    The script should:
    - Parse CLI arguments: `--mode=setup|teardown|test`, `--target=path/to/settings.json`, `--agent-chat-dir=/path/to/agent-chat`, `--project-dir=/path/to/target-project`
    - **Setup mode:**
      a. Read existing settings.json (or start with `{}`)
      b. Build AgentChat hook entries for: SessionStart, SessionEnd, PreToolUse, PostToolUse
         - Each hook command: `curl -s -X POST http://localhost:5555/api/hooks/{EventType} -H 'Content-Type: application/json' -d "$(cat)" 2>/dev/null || true`
         - SessionStart and SessionEnd: matcher is empty string `""`
         - PreToolUse and PostToolUse: matcher is `"*"`
      c. Build MCP server entry:
         ```json
         {
           "command": "node",
           "args": ["{agent-chat-dir}/packages/mcp/dist/index.js"],
           "env": {
             "AGENT_CHAT_DB_PATH": "{homedir}/.agent-chat/data.db",
             "AGENT_CHAT_TENANT_ID": "auto",
             "AGENT_CHAT_AGENT_NAME": "claude-agent",
             "AGENT_CHAT_CWD": "{project-dir}"
           }
         }
         ```
      d. Merge hooks: For each event type, check if AgentChat hook already exists (by checking if any hook command contains "localhost:5555/api/hooks"). If not, append the hook group to the event array. If the event array doesn't exist, create it.
      e. Merge mcpServers: If `agent-chat` key already exists, replace it (update paths). If not, add it.
      f. Write the merged JSON with 2-space indentation.

    - **Teardown mode:**
      a. Read existing settings.json (exit if doesn't exist)
      b. For each event type in hooks, filter out hook groups where any hook command contains "localhost:5555/api/hooks"
      c. Remove empty event arrays, remove empty hooks object
      d. Delete `agent-chat` key from mcpServers
      e. Remove empty mcpServers object
      f. Write back (or delete file if completely empty / only `{}`)

    - **Test mode (--test):**
      Run 6 built-in assertions:
      1. Setup on empty file creates correct structure
      2. Setup preserves existing hooks
      3. Setup preserves existing mcpServers
      4. Setup is idempotent (running twice produces same result)
      5. Teardown removes only AgentChat entries
      6. Teardown preserves other entries

      Each test: create temp file, run merge, verify JSON structure. Exit 0 if all pass, exit 1 with details if any fail.

    3. Make the script executable: `chmod +x scripts/lib/merge-settings.cjs`
  </action>
  <verify>
    <automated>cd /Users/dheer/code/personal/agent-chat && node scripts/lib/merge-settings.cjs --test 2>&1</automated>
  </verify>
  <done>
    - scripts/lib/merge-settings.cjs exists and is executable
    - Setup mode creates correct hooks and mcpServers entries
    - Teardown mode removes only AgentChat entries
    - Idempotent — running setup twice produces identical output
    - All 6 built-in self-tests pass
  </done>
